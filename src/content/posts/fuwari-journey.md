# 一个用 Fuwari 二开却逐渐失去理智的开发者自述

最近，我干了一件非常正常的事情：

用 Fuwari 项目二开了一个属于自己的博客。

听起来是不是很优雅？
像那种：

> "嗯，我最近在构建一个高性能、轻量级、优雅的个人内容平台。"

但实际上，我的开发过程是这样的：

---

## 一、项目启动：我以为我要做的是"博客"

刚 clone 下来那一刻，我内心只有四个字：

> ✨ 优雅！极致优雅！

毕竟 Fuwari 这种项目，本质就是一个——
**看起来很简单，但你一改就开始裂开**

然后我打开 VSCode，深吸一口气：

> "今天，我要成为一个真正的前端工程师。"

结果五分钟后：

- 样式炸了
- 主题错位
- 字体加载失败
- 页面白屏

我当场进入第一阶段心理状态：

👉 《创造美味佳肴中》

（实际上锅已经烧穿了）

---

## 二、二开阶段：我开始理解"没有告知的义务"

当我开始"微调一点点 UI"的时候，事情开始失控。

改一个 padding：

整个页面布局重排

改一个颜色变量：

深色模式直接变成灵堂模式

改一个组件：

build 报错三连击

此时我的内心逐渐变成：

NOT OBLIGATED TO INFORM
没有告知的义务

![没有告知的义务](../../assets/images/d5844d3254783ebb478fee57796b840e.jpg)

是的，代码不会告诉你哪里错了
它只会冷冷地说：

`TypeError: undefined is not a function`

你问它 undefined 是谁？

它：没有告知的义务

## 三、表情包系统：程序员的最终武器

当我已经写不动代码的时候，我做了一个决定：

上表情包。

于是我的博客逐渐变成：

首页：技术文章 ❌
首页：精神状态展示 ✔

比如：

当页面成功跑起来时：

我靠我真是天才

当 build 第 7 次失败时：

这辈子不可能打工的

当 CSS 再次炸裂时：

典型的我以为我会写前端

## 四、开发者的三种境界

写到现在，我总结出做这个网站的三个阶段：

**第一层：正常人**

- 我要写博客
- 我要记录生活
- 我要输出知识

**第二层：进阶开发者**

- 我需要优化 SEO
- 我要做主题切换
- 我要做组件抽象

**第三层：精神状态异常（我）**

- 为什么这个 margin 会影响 flex
- 为什么我删了一行代码整个项目挂了
- 为什么这个 bug 改完多了两个 bug

最终状态：

算了，加个表情包吧

## 五、最终成果

现在，我的网站：

| 项目  | 状态   |
| --- | ---- |
| 代码  | 一坨   |
| UI  | 还行   |
| 逻辑  | 勉强   |
| 表情包 | 极其丰富 |

总结一句话：

这个博客，不一定能教会你技术
但一定能让你知道一个程序员是怎么崩溃的

## 六、结语

如果你问我：

"你还会继续优化吗？"

我会坚定地回答：

👉 会。

因为程序员有一个终极信仰：

**再改一下就好了**
